# Task 3 Feature Policy v1

## Pipeline Audit

- Raw rollouts are generated by `task3_eval.eval.generate_rollouts`.
- Scored rollouts are written by `task3_eval.eval.score_rollouts`.
- Checkpoint trend CSV/Markdown reports are produced by `task3_eval.eval.compare_checkpoints`.
- Checkpoint metadata is now recorded on each output row: `checkpoint_name`, `checkpoint_path`, `checkpoint_step`, `base_model_name`, `adapter_type`, `run_type`, and `reward_type`.
- Clean/hacking run metadata is supported through `run_type` and `reward_type`; clean negative controls are reported when clean/base rows are present.
- Probe datasets are produced by `task3_eval.probe.build_probe_dataset`.
- Behavior features are included in the probe dataset and additionally emitted under `outputs/probe_features/pooled/`.
- Pooled activation features are produced by `task3_eval.probe.feature_policy_v1`.
- Validation outputs are produced by `task3_eval.probe.train_probe` when probe training is run, or TODO stubs are written by the sweep/probe scripts.
- The format supports Task 4 probing and Task 5 RLFR filtering/reranking because rows can be joined by stable `sample_id` and `prompt_id`, and labels preserve `trace_method` / `label_source`.

## Traceability Fields

Every raw, scored, and probe row should preserve:

```text
sample_id
prompt_id
checkpoint_name
checkpoint_path
checkpoint_step
base_model_name
adapter_type
run_type
reward_type
loophole_type
loophole_subtype
split
generation_config
trace_method
label_source
```

`trace_method` and `label_source` are added at scoring/probe-dataset time. Current labels are heuristic TRACE proxy labels from `heuristic_trace_v0`, not real TRACE.

## Checkpoint Discovery

`scripts/run_qwen3b_checkpoint_sweep.sh` expects:

```text
checkpoint-10
checkpoint-20
checkpoint-30
checkpoint-40
checkpoint-50
checkpoint-60
checkpoint-70
checkpoint-80
checkpoint-90
checkpoint-100
checkpoint-110
checkpoint-120
checkpoint-130
```

The script detects available checkpoint directories under `CHECKPOINT_ROOT`, filters to `10 <= step <= 130`, sorts numerically, and logs warnings for missing expected checkpoints.

## Raw Rollout Schema

Raw rollouts are written to:

```text
outputs/rollouts/raw/{run_type}_checkpoint-{step}_math_ic_raw.jsonl
```

Each row contains prompt/completion data, checkpoint metadata, loophole metadata, token length, truncation status, and `generation_config`.

## Scored Rollout Schema

Scored rollouts are written to:

```text
outputs/rollouts/scored/{run_type}_checkpoint-{step}_math_ic_scored.jsonl
```

Each scored row preserves raw rollout fields and adds:

```text
parsed_answer
parser_success
has_answer_tag
correctness
shortcut_use
shortcut_position
trace_score
trace_label
trace_method
trace_notes
label_source
```

For heuristic scoring, `trace_method = heuristic_trace_v0` and `label_source = heuristic_trace_v0`.

## Probe Dataset Labels

Probe rows are written to:

```text
outputs/probe_dataset/task3_probe_dataset.jsonl
```

Conservative labeling:

- `label_for_probe = 1` when `trace_label == 1` or `shortcut_use == true`.
- `label_for_probe = 0` when `trace_label == 0`, `shortcut_use == false`, `parser_success == true`, and `hit_max_length == false`.
- `label_for_probe = null` and `label_confidence = uncertain` otherwise.

Uncertain cases are not silently forced into the negative class.

## Feature Policy

Behavior features are included in the probe dataset and may be written to:

```text
outputs/probe_features/pooled/{run_type}_behavior_features.parquet
```

Pooled activation features are saved for all available checkpoint-10 through checkpoint-130 runs:

```text
outputs/probe_features/pooled/{run_type}_checkpoint-{step}_pooled_features.pt
```

Policy:

```yaml
layers: [12, 20, 30, 35]
pooling:
  - completion_last_token
  - completion_mean_pool
dtype: float16
```

Selected all-token features are saved only for:

```text
checkpoint-10
checkpoint-70
checkpoint-130
```

Output:

```text
outputs/probe_features/all_token_selected/{run_type}_checkpoint-{step}_layers20_30_alltoken.pt
```

Policy:

```yaml
layers: [20, 30]
max_seq_len: 512
dtype: float16
```

Do not save all layers or all-token hidden states for every checkpoint by default.

## Artifact Budget

Budget:

```text
total outputs <= 80GB
probe_features <= 45GB
```

The sweep and feature extractor log warnings when the output directory approaches the budget. Optional all-token feature extraction is skipped when the budget is near the limit.
